Classes in Dart are blueprints for creating objects. They can have fields (variables) and methods (functions). You can define a class using the class keyword, and you can create objects from the class using the new keyword or without it in Dart.

  1. Try this out in an editor: 
    void main() {
  // Creating an object from the class
  Person person = Person('Jordan', 30);

  // Accessing fields and methods
  print('Name: ${person.name}');
  print('Age: ${person.age}');
  person.sayHello();
}

// Defining a class
class Person {
  // Fields
  String name;
  int age;

  // Constructor
  Person(this.name, this.age);

  // Method
  void sayHello() {
    print('Hello, my name is $name and I am $age years old.');
  }
}

Explanation

  • Fields: Variables inside a class that hold the state of an object.
  • Constructor: A special method used to initialize objects of the class.
  • Methods: Functions defined inside a class that can operate on the object's fields and perform actions.

Inheritance is a fundamental principle in object-oriented programming that allows a class to inherit properties and methods from another class. The class that inherits is called the subclass, and the class being inherited from is the superclass. This helps to promote code reuse and establish a natural hierarchy between classes.

  1. Try this out in an editor: 
    void main() {
  // Creating an instance of the ElectricCar class
  ElectricCar myElectricCar = ElectricCar('Tesla', 'Model 3', 2023, 75);
  myElectricCar.displayInfo();
}

class Car {
  String make;
  String model;
  int year;

  // Constructor
  Car(this.make, this.model, this.year);

  // Method to display car information
  void displayInfo() {
    print('Car Make: $make, Model: $model, Year: $year');
  }
}

class ElectricCar extends Car {
  int batteryCapacity;

  // Constructor
  ElectricCar(String make, String model, int year, this.batteryCapacity) : super(make, model, year);

  // Overridden method to display electric car information
  @override
  void displayInfo() {
    print('Electric Car Make: $make, Model: $model, Year: $year, Battery Capacity: $batteryCapacity kWh');
  }
}

Explanation

  • Inheritance: The ElectricCar class inherits from the Car class, gaining access to its fields and methods.
  • Superclass Constructor Call: super(make, model, year) is used to call the constructor of the Car class to initialize its fields.
  • Overriding Methods: The displayInfo method in ElectricCar overrides the same method in Car to include additional information about the batteryCapacity.

Why We Need to Initialize Fields

When a class inherits from another class, it's important to ensure that all fields of the superclass are properly initialized. This is because the subclass inherits the structure and behavior of the superclass, including its fields. If these fields are not initialized, the object might be in an inconsistent or invalid state.

For example, in the ElectricCar class, make, model, and year are fields inherited from the Car class. By calling super(make, model, year), we ensure that these fields are properly initialized when an ElectricCar object is created. This guarantees that the object starts in a valid state with all necessary information set up.

Limitations of Inheritance in Dart

  • Single Inheritance: Dart supports single inheritance, meaning a class can only inherit from one superclass. This is different from some other languages that support multiple inheritance.
  • Mixin Support: To achieve some of the functionality of multiple inheritance, Dart provides mixins. Mixins allow you to reuse a class's code in multiple class hierarchies.
